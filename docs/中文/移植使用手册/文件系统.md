# lttit 文件系统使用手册（FS）

lttit 内置一个极简文件系统，支持目录、文件、读写、创建、遍历等基本功能。底层依赖块设备驱动（`fs_blkdev`），并使用位图管理 inode 和数据块。

本文主要介绍API的使用以及如何移植。

# 挂载与格式化

### **挂载文件系统**

```
int fs_mount(struct superblock *sb, struct fs_blkdev *bdev);
```

- 从块设备读取 superblock、inode bitmap、block bitmap、inode 表
- 初始化根目录
- 成功返回 0

### **卸载文件系统**

```
int fs_unmount(struct superblock *sb);
```

- 释放根 inode
- 调用底层 sync
- 成功返回 0

### **格式化文件系统**

```
int fs_format(struct superblock *sb);
```

- 清空位图
- 初始化 inode 表
- 创建根目录
- 写回 superblock、位图、inode 表
- 成功返回 0

# 目录操作

### **创建目录（递归创建）**

```
int fs_mkdir(const char *path, struct inode **out);
```

- 自动递归创建路径
- 若路径已存在，返回对应 inode
- 成功返回 0

示例：

```
struct inode *dir;
fs_mkdir("a/b/c", &dir);
```

### **读取目录内容**

```
int fs_readdir(const char *path, struct dirent *buf, int max, int *nread);
```

- 读取目录下的所有文件/子目录
- `buf` 为输出数组
- `max` 为最大数量
- `nread` 返回实际读取数量

示例：

```
struct dirent ents[16];
int n;
fs_readdir("a/b", ents, 16, &n);
```

# 文件操作

### **打开文件**

```
int fs_open(const char *path, int flags, struct inode **out);
```

支持：

- `O_CREAT`：不存在则创建
- `O_EXCL`：与 `O_CREAT` 一起使用时，若存在则失败
- `O_RDONLY / O_WRONLY / O_RDWR`（目前未严格区分）

示例：

```
struct inode *file;
fs_open("a/b/c.txt", O_CREAT, &file);
```

### **读取文件**

```
int fs_read(struct inode *inode, uint32_t off, void *buf, uint32_t len);
```

- 从文件偏移 `off` 读取 `len` 字节
- 返回实际读取字节数

示例：

```
char buf[128];
int n = fs_read(file, 0, buf, sizeof(buf));
```

### **写入文件**

```
int fs_write(struct inode *inode, uint32_t off, const void *buf, uint32_t len);
```

- 自动分配数据块
- 自动扩展文件大小
- 返回实际写入字节数

示例：

```
fs_write(file, 0, "hello", 5);
```

### **获取文件大小**

```
uint32_t fs_get_size(struct inode *inode);
```

### **关闭文件**

```
int fs_close(struct inode *inode);
```

- refcnt 归零后释放 inode
- 同步位图与 inode 表到块设备
- 成功返回 0

# 路径解析

文件系统支持简单路径解析：

- `"a/b/c"`：相对路径
- `"/"`：根目录
- 自动递归创建目录（`fs_mkdir`）
- `fs_open` 会自动创建父目录（若使用 `O_CREAT`）

## 底层块设备接口（必须实现）

用户必须提供一个块设备驱动：

```
struct fs_blkdev {
    uint32_t block_size;
    uint32_t block_count;
    uint32_t prog_size;
    uint32_t read_size;
    void *ctx;

    int (*read)(void *ctx, uint32_t blk, uint32_t off, void *buf, uint32_t len);
    int (*write)(void *ctx, uint32_t blk, uint32_t off, const void *buf, uint32_t len);
    int (*erase)(void *ctx, uint32_t blk);
    int (*sync)(void *ctx);
};
```

最小要求：

- `read`
- `write`
- `erase`（用于 flash）
- `sync`（可选）

# 文件系统限制（重要）

- **每个 inode 仅支持 8 个 direct 块（NDIRECT = 8）**
- **不支持间接块**
- **不支持删除目录（fs_rmdir 未实现）**
- **不支持 truncate**
- **不支持硬链接/软链接**
- **不支持权限模型**

这是一个极简 FS，适合嵌入式场景。



# 移植

# 文件系统移植（fs_port）

lttit 文件系统依赖一个抽象的块设备接口 `struct fs_blkdev`。
 移植的核心就是：**把底层 Flash（或其他存储）适配成 fs_blkdev**。

以 **STM32F1 内部 Flash** 为例，给出完整移植流程。

# 定义 Flash 区域

```
#define FS_FLASH_BASE 0x0800C000U
```

文件系统将从此地址开始使用 Flash 空间。
 用户可根据自己的工程修改起始地址。

# 实现底层读写擦除接口

文件系统需要 4 个底层函数：

```
read()
write()
erase()
sync()
```

### **读 Flash**

```
static int fs_bd_read(void *ctx, uint32_t block,
                      uint32_t off, void *buffer, uint32_t size)
{
    uint32_t addr = FS_FLASH_BASE + block * FS_BLOCK_SIZE + off;
    memcpy(buffer, (const void*)addr, size);
    return FS_ERR_OK;
}
```

### **写 Flash（半字写入）**

```
static int fs_bd_write(void *ctx, uint32_t block,
                       uint32_t off, const void *buffer, uint32_t size)
{
    HAL_FLASH_Unlock();
    ...
    HAL_FLASH_Lock();
    return FS_ERR_OK;
}
```

### **擦除 Flash（按页）**

```
static int fs_bd_erase(void *ctx, uint32_t block)
{
    FLASH_EraseInitTypeDef ei = {0};
    ...
    return (st == HAL_OK) ? FS_ERR_OK : FS_ERR_IO;
}
```

### **同步（可选）**

```
static int fs_bd_sync(void *ctx) { return FS_ERR_OK; }
```

# 初始化块设备结构体

```
struct fs_blkdev fs_dev;

void fs_port_init(void)
{
    g_bdev = &fs_dev;
    memset(&fs_dev, 0, sizeof(fs_dev));

    fs_dev.read  = fs_bd_read;
    fs_dev.write = fs_bd_write;
    fs_dev.erase = fs_bd_erase;
    fs_dev.sync  = fs_bd_sync;

    fs_dev.read_size   = FS_READ_SIZE;
    fs_dev.prog_size   = FS_PROG_SIZE;
    fs_dev.block_size  = FS_BLOCK_SIZE;
    fs_dev.block_count = FS_BLOCK_COUNT;
}
```

这一步把 Flash 适配成文件系统可识别的块设备。

# 完整使用示例

```
struct superblock sb;

int main(void)
{
    HAL_Init();
    SystemClock_Config();

    fs_port_init();

    if (fs_port_mount(&sb) != 0) {
        printf("FS mount failed\n");
        while (1);
    }

    printf("FS mounted!\n");

    struct inode *ino;
    fs_open("root/hello.txt", O_CREAT, &ino);
    fs_write(ino, 0, "Hello FS!\n", 10);
    fs_close(ino);

    fs_port_deinit(&sb);

    while (1);
}
```

# 移植总结

要移植文件系统，只需要：

1. **确定 Flash 区域（FS_FLASH_BASE）**
2. **实现 read/write/erase/sync**
3. **初始化 fs_blkdev**
4. **调用 fs_port_mount()**
5. **正常使用 FS API（open/read/write/close）**
