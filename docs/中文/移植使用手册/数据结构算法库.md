# 数据结构和算法库

包含链表、队列、栈、红黑树、哈希表、radix树、一些数学运算库。

# 双向循环链表（link_list）

这是一个**最小化实现的双向循环链表**，适合作为各种容器（队列、栈、对象池等）的基础结构。

### **API**

```
void list_node_init(struct list_node *node);     // 初始化节点（自环）
int  list_empty(struct list_node *node);         // 判断链表是否为空

void list_add_prev(struct list_node *next,
                   struct list_node *prev);      // prev 插入到 next 前

void list_add_next(struct list_node *prev,
                   struct list_node *next);      // next 插入到 prev 后

void list_remove(struct list_node *node);        // 从链表中移除节点
```

### **使用方法**

```
struct list_node head;
list_node_init(&head);

struct list_node n1;
list_node_init(&n1);

list_add_next(&head, &n1);   // 插入
list_remove(&n1);            // 删除
```

### **特性**

- **循环链表**：head 的 next/prev 指向自身。
- **无内存分配**：节点由用户自行管理。
- **O(1) 插入/删除**：适合高性能场景。
- **可作为队列、栈、对象池等基础结构**。

# 队列（queue）

基于链表实现的 **FIFO 队列**。

### **API**

```
void queue_enqueue(struct list_node *head,
                   struct list_node *node);   // 入队（尾部）

struct list_node *queue_dequeue(struct list_node *head); // 出队（头部）
```

### **使用方法**

```
struct list_node q;
list_node_init(&q);

struct list_node n;
list_node_init(&n);

queue_enqueue(&q, &n);
struct list_node *x = queue_dequeue(&q);
```

### 特性

- **FIFO**：先进先出。
- **基于链表**：无容量限制。
- **O(1) 入队/出队**。

# 栈（stack）

基于链表实现的 **LIFO 栈**。

### **API**

```
void stack_push(struct list_node *head,
                struct list_node *node);   // 入栈（头部）

struct list_node *stack_pop(struct list_node *head); // 出栈（头部）
```

### **使用方法**

```
struct list_node s;
list_node_init(&s);

struct list_node n;
list_node_init(&n);

stack_push(&s, &n);
struct list_node *x = stack_pop(&s);
```

## **特性**

- **LIFO**：后进先出。
- **基于链表**：无容量限制。
- **O(1) push/pop**。

### 小结

| 模块      | 行为         | 时间复杂度     | 特点                 |
| --------- | ------------ | -------------- | -------------------- |
| link_list | 双向循环链表 | O(1) 插入/删除 | 所有容器的基础       |
| queue     | FIFO 队列    | O(1)           | 基于链表，无容量限制 |
| stack     | LIFO 栈      | O(1)           | 基于链表，无容量限制 |

# 哈希表（hashmap）

哈希表实现，采用 **链式哈希（separate chaining）**，支持字符串、整数和指针作为键，使用黄金比例作为哈希算法。桶数量自动向上取 2 的幂，冲突通过双向链表解决。

### **API**

```
void hashmap_init(struct hashmap *map, size_t bucket_count, int key_type);
void hashmap_put(struct hashmap *map, void *key, void *value);
void *hashmap_get(struct hashmap *map, void *key);
int  hashmap_remove(struct hashmap *map, void *key);
int  hashmap_contains(struct hashmap *map, void *key);
void hashmap_clear(struct hashmap *map);
void hashmap_destroy(struct hashmap *map);
```

### **使用方法**

```
struct hashmap map;

hashmap_init(&map, 16, HASHMAP_KEY_STRING);

hashmap_put(&map, "hello", (void*)123);
void *v = hashmap_get(&map, "hello");

hashmap_remove(&map, "hello");
hashmap_destroy(&map);
```

## 特性

- **链式哈希表**：每个 bucket 是一个双向链表。
- **自动对齐桶数量**：`bucket_count` 会自动提升为最近的 2 的幂。
- 三种 key 类型：
  - `HASHMAP_KEY_STRING`：字符串键，使用自定义字符串哈希。
  - `HASHMAP_KEY_INT`：整数键（直接使用值作为 key）。
  - `HASHMAP_KEY_PTR`：指针键（按地址比较）。
- **冲突处理**：链表插入，查找为 O(n)（n 为桶内元素数）。
- **无锁**：适用于单核 MCU，如需并发请自行加锁。
- **自动内存管理**：entry 使用 `heap_malloc` 分配，`hashmap_destroy` 会释放所有 entry。

## 注意

- 字符串键不会复制，必须保证 key 的生命周期有效。
- `hashmap_destroy` 会释放所有 entry，但不会释放 key/value 指向的对象。
- 若需要线程安全，请在外层加锁。
- bucket 数量越大，冲突越少，性能越好。
- 当key相似时，哈希冲突可能比较严重，这种情况下可以使用radix树。

# Radix Tree（基数树）

Radix Tree 是一种按位前缀组织的多叉树结构，适合管理稀疏整数索引。
 支持快速插入、查找、删除，以及上界查找（upper_bound）。

本实现按 **BIT_LEVEL = 4** 划分，每层 16 个子节点，可以通过宏自由调整。

## **API**

```
void radix_tree_init(struct radix_tree_root *root);                 // 初始化
int  radix_tree_insert(struct radix_tree_root *root, size_t index, void *item); 
void *radix_tree_lookup(struct radix_tree_root *root, size_t index); 
void *radix_tree_lookup_upper_bound(struct radix_tree_root *root, size_t index);
void *radix_tree_root_left(struct radix_tree_root *root);           // 最小元素
void *radix_tree_delete(struct radix_tree_root *root, size_t index);
```

## **使用方法**

```
struct radix_tree_root root;
radix_tree_init(&root);

radix_tree_insert(&root, 128, ptr);
void *p = radix_tree_lookup(&root, 128);

void *q = radix_tree_lookup_upper_bound(&root, 100); // >=100 的最小元素

radix_tree_delete(&root, 128);
```

## 特性

- **按位前缀分层**：每层消费 `BIT_LEVEL` 位（默认 4 位 → 16 路树）。
- **高度自适应**：根据 index 的最高位自动扩展树高。
- **稀疏索引友好**：不需要连续空间，适合稀疏 key。
- **支持 upper_bound**：可用于“找最小可用块”等场景。
- **自动节点回收**：删除后若节点为空，会向上回收。
- **无锁**：适用于单核 MCU，如需并发请自行加锁。

##  Radix Tree与哈希表对比

- key **相似、集中、有结构** → hash 冲突严重
- key **需要顺序** → hashmap 不支持
- key **需要 upper_bound** → hashmap 做不到
- key **稀疏** → 数组浪费空间

Radix Tree 在这些场景下比哈希表更稳定、更可控。

## **注意**

- index 必须是非负整数。
- BIT_LEVEL 可调（1/2/4/8），影响树的宽度与深度。
- 插入同一个 index 会失败（返回 -1），不过，你可以通过保存指针的方法解决冲突，类似于链式哈希表。
- 删除后树会自动收缩。

# 红黑树（rbtree）

红黑树是一种自平衡二叉搜索树，支持按 value 有序插入、查找、删除，并提供前驱/后继遍历能力。本实现基于 Linux 内核风格，额外维护了 `first_node` 与 `last_node`，可快速获取最小/最大节点。

## **API**

```
void rb_root_init(rb_root_handle root);                 // 初始化根节点
void rb_node_init(rb_node_handle node);                 // 初始化节点
void rb_insert_node(rb_root_handle root, rb_node_handle new_node); 
void rb_remove_node(rb_root_handle root, rb_node_handle node);

rb_node_handle rb_first(rb_root_handle root);           // 最小节点
rb_node_handle rb_last(rb_root_handle root);            // 最大节点
rb_node_handle rb_next(rb_node_handle node);            // 后继
rb_node_handle rb_prev(rb_node_handle node);            // 前驱
rb_node_handle rb_first_greater(rb_root_handle root, size_t key); // >key 的最小节点

void rb_replace_node(rb_node_handle victim,
                     rb_node_handle new_node,
                     rb_root_handle root);
```

## **使用方法**

```
struct rb_root root;
rb_root_init(&root);

struct rb_node node;
rb_node_init(&node);
node.value = 100;

rb_insert_node(&root, &node);

struct rb_node *n = rb_first(&root);   // 最小节点
rb_remove_node(&root, &node);
```

## **特性**

- **按 value 排序**：插入时根据 `node->value` 进行二叉搜索树定位。
- **自动平衡**：插入/删除后自动旋转与染色，保持树高度稳定。
- **维护 first/last**：插入时更新 `first_node` / `last_node`，O(1) 获取最小/最大节点。
- **支持前驱/后继遍历**：`rb_next` / `rb_prev` 可按排序顺序遍历。
- **支持 upper_bound**：`rb_first_greater(root, key)` 返回第一个 value > key 的节点。
- **无锁**：适用于单核 MCU，如需并发请自行加锁。

## **注意**

- 节点由用户自行分配（通常在结构体中嵌入 `struct rb_node`）。
- `value` 必须由用户设置，用于排序。
- 不允许插入重复 value（否则会覆盖排序结构）。
- 删除节点后，节点内容不会被清空，用户可自行复用。

## **适用场景**

- 内存分配器（按 block_size 排序）
- 定时器（按 expire_time 排序）
- 调度器（按 deadline / respond_line 排序）
- 文件系统索引
- 任意需要 ordered set / ordered map 的场景
