# TitVim：内置文本编辑器使用手册

TitVim 是 lttit 系统内置的轻量级文本编辑器，支持：

- 打开/创建文件
- 插入模式
- 删除、换行
- 保存、退出
- 简单的光标移动
- 与内置文件系统（FS）无缝集成

它运行在串口终端上，通过 `comm_getc()` / `comm_write()` 与用户交互。

# 启动编辑器

```
vim_main("/path/to/file");
```

行为：

- 若文件存在 → 加载内容
- 若文件不存在 → 创建空 buffer
- 光标初始位置为 (0,0)

# 基本界面

编辑器启动后会：

- 清屏
- 显示所有行
- 在底部显示 `-- INSERT --`（插入模式）
- 光标位置实时更新

# 模式

TitVim 有两种模式：

## **普通模式（Normal Mode）**

按下 `ESC` 进入普通模式。

支持：

| 按键 | 功能                  |
| ---- | --------------------- |
| `i`  | 进入插入模式          |
| `h`  | 光标左移              |
| `l`  | 光标右移              |
| `j`  | 光标下移              |
| `k`  | 光标上移              |
| `x`  | 删除光标处字符        |
| `:`  | 输入命令（保存/退出） |

## 插入模式（Insert Mode）

按下 `i` 进入插入模式。

支持：

| 按键        | 功能           |
| ----------- | -------------- |
| 普通字符    | 插入到光标处   |
| `Enter`     | 换行           |
| `Backspace` | 删除前一个字符 |
| `ESC`       | 返回普通模式   |

# 命令模式（:）

在普通模式下按 `:` 进入命令输入。

支持的命令：

| 命令 | 功能       |
| ---- | ---------- |
| `:w` | 保存文件   |
| `:q` | 退出编辑器 |

示例：

```
:w
:q
```

# 文件加载（buf_load）

```
int buf_load(struct buffer *b, const char *path);
```

行为：

- 调用 `fs_open()` 打开文件
- 使用 `fs_read()` 读取内容
- 按行切分
- 若文件为空 → 创建一行空行
- 若文件不存在 → 创建空 buffer

# 文件保存（buf_save）

```
int buf_save(struct buffer *b, const char *path);
```

行为：

- 调用 `fs_open(path, O_CREAT)`
- 将所有行拼接成文本（每行追加 `\n`）
- 调用 `fs_truncate()` 清空旧内容
- 调用 `fs_write()` 写入新内容

# 编辑操作

### **插入字符**

```
buf_insert_char(&buf, c);
```

自动扩容行缓冲区。

### **删除字符（Backspace）**

```
buf_backspace(&buf);
```

支持：

- 行内删除
- 行合并（光标在行首时）

### **删除字符（x）**

```
buf_delete_char(&buf);
```

删除光标处字符。

### **换行**

```
buf_newline(&buf);
```

将当前行拆分为两行。

# 内存管理

编辑器使用你的 heap：

- 每行 `char*` 动态分配
- 行数组动态扩容
- 保存/加载时使用临时 heap buffer
- 退出时调用：

```
buf_free(&buf);
```

释放所有行。

# 退出行为

执行 `:q` 时：

- 清空屏幕
- 绘制一个空 buffer（保持终端整洁）
- 输出换行
- 释放所有内存
- 返回到调用者

# 最小使用示例

```
void shell_vim(const char *path)
{
    vim_main(path);
}
```

在你的系统里，这通常绑定到 shell 命令：

```
> vim /etc/config.txt
```

# 特点与限制

- 不支持光标跨行移动时自动调整列
- 不支持撤销/重做
- 不支持搜索
- 不支持语法高亮
- 不支持多文件
- 不支持可视模式

但它足够：

- 编辑配置文件
- 查看日志
- 修改脚本
- 在 MCU 上直接编辑 Flash 文件系统内容
